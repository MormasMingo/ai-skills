# 检查点管理规范

## 1. 概述

### 1.1 目的
规范检查点文档的创建、更新和使用流程，确保任务进度的可追溯性。

### 1.2 适用范围
适用于所有项目的任务管理和进度跟踪。

## 2. 检查点文档结构

### 2.1 元信息
```yaml
---
checkpoint-id: YYYY-MM-DD-[序号]
date: YYYY-MM-DD
status: active|completed|archived
---
```

### 2.2 四个标准部分

#### 第一部分：禁止行为（必须严格遵守）
- 列出项目中绝对禁止的行为
- 使用 `[ ]` 标记，便于检查

#### 第二部分：建议规范（建议遵守）
- 分类列出建议遵守的规范
- 使用 `[ ]` 标记

#### 第三部分：待办任务（本次需求）
分为三个子部分：
- **已完成** - 使用 `[x]` 标记
- **进行中** - 使用 `[ ]` 标记
- **待开始** - 使用 `[ ]` 标记

#### 第四部分：后续备忘（暂不处理）
- 记录暂时不处理但重要的事项
- 定期回顾

## 3. 使用流程

### 3.1 创建时机
- 新需求开始时
- 复杂任务分解时
- 对话上下文丢失风险时

### 3.2 查阅时机
- 每次对话开始前
- 对话过长或被压缩后

### 3.3 更新时机
- 任务状态变更时立即更新
- 发现新的禁止行为或建议规范时
